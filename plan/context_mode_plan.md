# 📋 План внедрения: режим "Слово в контексте"

> Новый режим обучения — пользователь видит пример с пропуском и выбирает правильное слово из 4 вариантов. Цель: переход от пассивного запоминания к использованию слов в контексте.

---

## 🎯 Суть режима

```
Пример:  "Er _____ jeden Tag zur Arbeit."
         (ходить пешком)

Варианты:
  A) geht       ← правильно
  B) schläft
  C) kauft
  D) schreibt
```

- Показывается сгенерированный пример с пропуском вместо слова
- Под пропуском — перевод слова как якорь (защита от синонимов)
- 4 варианта: 1 правильный + 3 дистрактора той же части речи

---

## 🗂️ Этапы внедрения

---

### Этап 1 — Обновление генерации примеров
**Оценка: 2–3 дня**

Текущая логика генерирует примеры при добавлении карточек. Нужно обновить промпт под требования нового режима.

#### 1.1 Обновить промпт для Gemini

Новые требования к примеру:
- Слово стоит в **начальной форме** (инфинитив, единственное число) — чтобы совпадало с карточкой
- Контекст **не раскрывает** значение без знания слова
- Слово — **ключевое** в предложении, не периферийное
- Длина: **8–12 слов**
- Рядом с пропуском **нет артикля** (актуально для немецкого)
- Предложение **бытовое и реалистичное**

```
Промпт:

Создай одно предложение на [язык] со словом "{word}" ({translation}).

Требования:
1. Используй слово в базовой форме: инфинитив для глаголов, 
   именительный падеж ед.ч. для существительных
2. Слово должно быть ключевым — без знания его значения 
   смысл предложения непонятен
3. Не ставь артикль непосредственно перед словом
4. Длина предложения: 8–12 слов
5. Предложение бытовое, реалистичное
6. Другие слова с похожим значением ({synonyms_if_known}) 
   НЕ должны подходить по смыслу в этом контексте

Верни ТОЛЬКО предложение, без пояснений.
```

#### 1.2 Хранение примеров

Проверить что в модели карточки есть поле `example`. Оно уже существует в `ApiService.ts`. Никаких изменений схемы не требуется.

#### 1.3 Fallback для старых карточек

Если у карточки нет примера — генерировать **на лету** при первом открытии режима:

```typescript
// При загрузке карточек для режима
const cardsWithExamples = await Promise.all(
  cards.map(async (card) => {
    if (!card.example) {
      card.example = await generateExampleOnDemand(card);
    }
    return card;
  })
);
```

Показывать лоадер: *"AI готовит задания..."* — аналогично текущему экрану генерации.

### Этап 2 — Логика подбора дистракторов
**Оценка: 1–2 дня**

#### 2.1 Алгоритм подбора (приоритет по уровням)

```
Уровень 1: карточки из того же набора, той же части речи
           → нужно минимум 3 штуки

Уровень 2: если не хватает → карточки из других наборов 
           пользователя, той же части речи

Уровень 3: если всё равно не хватает → AI генерирует 
           подходящие слова-дистракторы
```

#### 2.2 Фильтрация по части речи

Использовать существующий тег `wordType` (noun / verb / adjective / other) для фильтрации. Дистракторы должны совпадать по `wordType` с правильным ответом.

```typescript
function getDistractors(
  correctCard: Card,
  allCards: Card[],
  count: number = 3
): Card[] {
  // Шаг 1: из того же набора, та же часть речи
  const sameSetSameType = allCards.filter(c =>
    c.id !== correctCard.id &&
    c.setId === correctCard.setId &&
    c.wordType === correctCard.wordType
  );

  if (sameSetSameType.length >= count) {
    return shuffle(sameSetSameType).slice(0, count);
  }

  // Шаг 2: из других наборов, та же часть речи
  const otherSetsSameType = allCards.filter(c =>
    c.id !== correctCard.id &&
    c.setId !== correctCard.setId &&
    c.wordType === correctCard.wordType
  );

  const combined = [...sameSetSameType, ...otherSetsSameType];
  if (combined.length >= count) {
    return shuffle(combined).slice(0, count);
  }

  // Шаг 3: AI генерирует дистракторы
  return generateAIDistractors(correctCard, count - combined.length);
}
```

#### 2.3 AI-генерация дистракторов (Уровень 3)

```
Промпт:

Дано слово "{word}" ({translation}), часть речи: {wordType}.
Придумай {n} слова на {язык} которые:
- Той же части речи
- Похожей тематики или сложности
- Явно отличаются по значению от "{word}"

Верни только слова через запятую, без пояснений.
```

**Оценка: 2–3 дня**

#### 3.3 Состояния кнопок после ответа

- **Правильно:** выбранная кнопка → зелёная, остальные серые, +1 балл
- **Неправильно:** выбранная → красная, правильная → зелёная
- Через **1.5 секунды** → автоматически следующая карточка

#### 3.4 Минимальный порог карточек

Если в наборе меньше **4 карточек** — показывать предупреждение:

```
"Для этого режима нужно минимум 4 карточки в наборе"
```

*(при 4+ карточках дистракторы уже можно подобрать, остальное добирается из других наборов)*

### Этап 4 — Статистика
**Оценка: 1 день**

#### 4.1 Отдельный счётчик

Результаты этого режима **не смешивать** с SRS классического режима. Хранить отдельно:

```typescript
interface ContextModeResult {
  cardId: string;
  correct: boolean;
  timestamp: Date;
  mode: 'context_fill'; // отличать от 'flashcard', 'multiple_choice'
}
```

#### 4.2 Новая метрика в статистике

На экране статистики добавить:

| Метрика | Описание |
|---------|----------|
| Угадано в контексте | Слова правильно выбранные в этом режиме |
| Точность | % правильных ответов |
| Прогресс | Сколько слов прошли оба режима (классика + контекст) |

### Этап 5 — Тестирование
**Оценка: 1–2 дня**

#### Чеклист перед релизом

- [ ] Набор с < 4 карточками → показывается предупреждение
- [ ] Набор с 4–9 карточками → дистракторы добираются из других наборов
- [ ] Карточки без примеров → генерируется на лету, лоадер показывается
- [ ] Синонимы в наборе → перевод-якорь снимает неоднозначность
- [ ] Немецкие существительные → артикль не стоит рядом с _____
- [ ] Правильный ответ рандомно на разных позициях (A/B/C/D)
- [ ] После ответа: 1.5 сек задержка перед следующей карточкой
- [ ] Результаты не влияют на SRS интервалы

## ⚠️ Известные ограничения

**Угадывание по грамматике** — частично неизбежно. Если дистракторы той же части речи, риск снижается, но не исчезает полностью. Это приемлемо для первой версии.

**Качество AI примеров** — промпт снижает риск, но не даёт 100% гарантии. Рекомендуется добавить возможность пользователю вручную редактировать пример карточки.
